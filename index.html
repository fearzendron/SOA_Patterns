Canonical Protocol

Dado que serviços podem aplicar diferentes tecnologias de comunicação e assim aumentar a complexidade de integração entre sistemas diferentes, o padrão Canonical Protocol sugere a aplicação de uma unica tecnologia para a comunicação.

Exemplo: SOAP sobre HTTP. Ou RESTful.

Canonical Schema

Diferentes serviços com modelos semelhantes acabam impondo um esforço de transformação nos dados. Para resolver este ponto este padrão sugere o uso de schemas entre serviços chamados de contratos.

Exemplo: Arquivos XML como o XSD para modelos de dados e WSDL para contratos.

Links:

  • soapatterns.org
  • wikipedia

Domain Inventory

Centralizar o inventorio de serviços pode trazer alta complexidade e muito esforço de todos os envolvidos em manter tudo organizado. Pessoas com papéis diferentes podem acabar demandando constante alinhamento com o time por causa disso. Para resolver este problema este pattern sugere a descentralização dos repositorios de serviços organizados por domínios.

Exemplo: Na pratica podemos criar repositorios de serviço por dominio, por exemplo, contabilidade, RH, PCP, vendas e etc.. Cada um gerenciado de forma independente.
